\section[课题的研究意义、国内外研究现状]{课题的研究意义、国内外研究现状}
\begin{frame}
\hilite<1> \frametitle{课题的研究意义}\pause
\begin{itemize}
\begin{spacing}{2}
\hilite<2>\item 薛定谔方程揭示了微观粒子的状态随时间变化的规律，不仅具有明确的量子力学意义，并且在很多的物理分支及其他科学领域得到了广泛应用。\pause
\hilite<3>\item 目前分数阶微分方程的研究已成为一个新的活跃研究领域，引起了国内外学者的高度重视。但大多数分数阶微分方程解析解的求解是十分困难的，因此研究相应的数值算法显得极为重要。\pause
\hilite<4>\item 随着科学技术和数学理论的不断发展，在各种科学领域中出现了各种各样的非线性问题，大量微分方程成为处理非线性问题的有力工具。而非线性薛定谔方程是非线性偏微分方程的一种，由于该方程的精确解难以得到，往往采用一些数值方法进行求解，但很多数值方法并不能够很好地满足守恒性。\pause
\hilite<5>\item 因此,研究非线性分数阶薛定谔方程高效的数值保守格式具有重要的理论意义和应用价值。\pause
\end{spacing}
\end{itemize}
\end{frame}
%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%
\begin{frame}
\frametitle{国内外研究现状}
\begin{itemize}
\begin{spacing}{2}
\hilite<1>  \item\color{black} 目前，学者们已经发展了众多守恒的数值方法，包括：有限差分法[6]、有限元法[7]、和谱方法[8,9]等守恒型数值方法。由于有限差分方法在实现的过程中比较方便，因此国内外的很多学者发展了多种形式的有限差分方法来求解非线性薛定谔方程。
\end{spacing}
\end{itemize}
\end{frame}

%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%
\begin{frame}
\hilite<1> \frametitle{国内外研究现状}\pause
\begin{itemize}
\begin{spacing}{2}
\hilite<2>\item 首先，对于经典薛定谔方程，在过去的几十年里，人们利用有限差分方法对其进行了大量的研究。\pause
\hilite<3>\item Bao等[10]提出了一个能量守恒的非线性 Crank-Nicolson 有限差分格式和一个半隐式有限差分格式和研究均匀误差估计。\pause
\hilite<4>\item Li等[11]构造一个紧凑的差分方案。\pause
\hilite<5>\item Wang等[12]给出离散时间正交样条配置方法。\pause
\hilite<6>\item Guo等[13]开发了一个能量守恒局部不连续伽辽金方法。\pause
\end{spacing}
\end{itemize}
\end{frame}
%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%

\begin{frame}
\hilite<1> \frametitle{国内外研究现状}\pause
\begin{itemize}
\begin{spacing}{2}
\hilite<2>\item 随着分数阶计算技术的发展，研究者越来越重视分数阶模型的研究。\pause
\hilite<3>\item Ran和Zhang[14]为该模型开发了一种三级线性隐式保守差分方案。\pause
\hilite<4>\item Cheng和Wu[15]为同一模型提出了几种保守紧差分方案。\pause
\hilite<5>\item Wang等人提出了非线性分数阶薛定谔方程的保能差分格式，见[16-18]。\pause
\end{spacing}
\end{itemize}
\end{frame}

%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%


\begin{frame}
\frametitle{国内外研究现状}
\begin{itemize}
\begin{spacing}{2}
\hilite<1>  \item 然而，上述的大多数节能方案要么是完全隐式的，要么是多层的，低精度的。\pause
\hilite<2>  \item 少数建立了一种高阶精度的线性节能方案。\pause
\hilite<3>  \item 鉴于此，我们希望的是基于近年来提出的标量辅助变量(SAV)方法[19-23]，针对非线性分数阶薛定谔方程提出一种高阶精度的节能线性数值方案。\pause
\end{spacing}
\end{itemize}
\end{frame}

\section[课题研究目标、研究内容、拟解决的关键问题]{课题研究目标、研究内容、拟解决的关键问题}
\begin{frame}
\hilite<1> \frametitle{课题研究目标}
\pause
\begin{itemize}
\begin{spacing}{2}
\hilite<1>  \item 首先，对非线性分数阶薛定谔方程，利用标量辅助变量（SAV）方法和有限差分方法构造高精度的保结构线性数值格式，并对提出格式进行其收敛性证明和数值验证； \pause
\hilite<2>  \item 其次，再利用SAV方法和有限差分方法构造出耦合方程的高精度线性数值格式，并对相应数值格式的收敛性进行分析和数值测试。\pause
\end{spacing}
\end{itemize}
\end{frame}

%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%
\begin{frame}
\hilite<1> \frametitle{课题研究内容}\pause
我们考虑如下的分数阶薛定谔方程：
\begin{align}
u_{t t}+(-\Delta)^{\alpha/2}u+\mathrm{i}u_{t}+\beta f(|u|^{2}) u=0,~x \in \mathbb{R},t \in(0, T],\\
u(x,0)=\phi_0(x),u_t(x,0)=\phi_1(x),~x \in \mathbb{R},\\
u(x,t)=0, u_t(x,t)=0,~x \in \mathbb{R}, t \in[0, T],
\end{align}
其中 $i=\sqrt{-1}, 1<\alpha\leq 2$, 和 $\beta$ 是正常数， $\phi _{0}(x)$ 和 $\phi _{1}(x)$都是给定的足够光滑的函数。$u(x, t)$是待确定的复波函数，非线性项f是一个给定的实函数。$|u|^{2}$ 或 $|u|^{4}$, $-(-\Delta)^{\alpha/2}u$ 表示Riesz分数阶导数,其定义参见[1]。
\end{frame}
%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%
%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%
\begin{frame}
\hilite<1> \frametitle{课题研究内容}
\pause
\begin{itemize}
\begin{spacing}{2}
\hilite<1>  \item 首先，我们拟对非线性分数阶薛定谔方程，基于SAV方法，四阶中心算子和Crank-Nicolson方法建立保结构线性数值格式,然后对该数值格式的收敛性进行严格的理论分析。\pause
\hilite<2>  \item 其次，拟再利用SAV方法和有限差分方法构造出耦合方程的高精度线性数值格式，分析其收敛性。\pause
\hilite<3>  \item 应用数值算例证明我们所提出的数值格式是可行的，能与理论研究结果保持一致。
\end{spacing}
\end{itemize}
\end{frame}

%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%
\begin{frame}
\hilite<1> \frametitle{拟解决的关键问题}
\pause
\begin{itemize}
\begin{spacing}{2}
\hilite<1>  \item 首先，如何把SAV方法应用到分数阶非线性薛定谔方程，对非线性项进行显式计算，构造线性保结构格式。 \pause
\hilite<2>  \item 其次，对所构造的数值格式收敛性的理论分析。\pause
\hilite<3>  \item 最后，考虑到问题的复杂性，对相应算法的程序实现和数值模拟工作。\pause
\end{spacing}
\end{itemize}
\end{frame}
